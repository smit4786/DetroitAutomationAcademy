DATA FLOW SUMMARY
================================================================================

1. Attendee scans QR code (poster/sign/card)
2. Opens GitHub enrollment form in browser
3. Fills out and submits form
4. GitHub creates issue with "enrollment" label
5. Dashboard fetches issues via API (every 30s)
6. Calculates statistics from issue data
7. Displays on staff terminal (detailed) + large screen (key metrics)
8. Notifies staff when milestones reached

================================================================================
KEY CHARACTERISTICS
================================================================================

âœ… Zero Interference - QR codes and dashboard work independently
âœ… Read-Only - Dashboard never modifies enrollment data
âœ… Real-Time - 30-second refresh interval
âœ… Dual Display - Staff analytics + attendee visibility
âœ… Milestone Alerts - Celebrates achievements as they happen
âœ… Secure - Token-based authentication, revocable access

   - Works with existing laptop/projector setup
   - Requires only internet connection
