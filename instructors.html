üéØ Quick Start for Teachers

1. Setup & Preparation

  • Read the Quick Start guide (15 minutes)
  • Install Python 3.8+ and required packages
  • Test GPIO examples with Raspberry Pi (or mock GPIO for dev)
  • Set up 3D printer and laser cutter access

2. Teaching Approach

Hands-On + Pedagogical

Each phase combines:

  • Learning Objectives: Clear outcomes for each session
  • Live Coding: Demo + code-along for students
  • Debugging Challenges: Intentional bugs to debug (great teaching tool!)
  • Real Hardware: Hands-on work with actual robots and equipment
  • Projects: Build toward a final portfolio piece each phase

3. Assessment & Grading

We use a portfolio-based approach:

  • Phase Completion: Successfully run all 3 phases (40%)
  • Debugging Challenges: Solve intentional bugs correctly (30%)
  • Final Project: Integrated project using all 3 phases (30%)
  • Attendance: 10+ of 12 sessions required (prerequisite)

‚ùì Instructor FAQ

How much prep time do I need each week?

Most instructors spend 1-2 hours prepping per session. Much of it is reviewing the phase guide and testing code examples. After teaching phase 1-2 times, it becomes faster!

What if a student gets stuck?

Our debugging challenges are intentionally designed to teach problem-solving. Encourage students to:

  • Read error messages carefully
  • Use print statements to debug
  • Check hardware connections
  • Ask peers for help (collaborative learning)

Can I customize the curriculum?

Absolutely! The phases are structured, but you can:

  • Add additional challenges or projects
  • Adjust pacing to match your cohort's pace
  • Bring in guest speakers or industry partnerships
  • Share improvements back to the main project

How do I report issues or suggest improvements?

Open a GitHub issue or contact us at dbkrsmith+daa@gmail.com. We actively incorporate teacher feedback!
